# Map Notes (09_Maps)

A map-of-content note is the one-page answer to "what is going on in this project?" —
generated by MindVault, readable by humans in Obsidian, and the cheapest orientation read
an agent can make.

## Commands

```bash
mindvault map create --project "MindVault"     # first time
mindvault map rebuild --project "MindVault"    # refresh the generated block
mindvault map list
```

MCP: `mindvault_create_map`, `mindvault_rebuild_map`, `mindvault_list_maps`.

Maps live at `09_Maps/<Project> Map.md` and carry `type: map` — deliberately **not** a
managed type: they are generated artifacts, not memory, so validation does not police
them and orphan/duplicate logic ignores them.

## What a map contains

```markdown
# MindVault Map

<!-- mindvault-generated:start -->
## Project Hub
- [[MindVault]]

## Current Goal
(pulled from the hub's Goal section)

## Key Decisions
- [[Decision - Use SQLite FTS5]] — accepted (2026-07-01)

## Active Tasks
## Open Risks
## Known Mistakes
## Constraints
## Recent Implementation Logs
## Reviews
## Related Prompts
## Orphaned / Needs Review
## Last Rebuilt
2026-07-04 14:00 — `mindvault map rebuild --project "MindVault"`
<!-- mindvault-generated:end -->
```

Every entry is a `[[wiki link]]` with status and date — compact refs, never note bodies.
Archived notes are excluded. Sections are bounded (10 entries for decisions/tasks/risks/
mistakes/constraints, 5 for logs/reviews/prompts/orphans).

## The generated-block contract

- Everything between `<!-- mindvault-generated:start -->` and
  `<!-- mindvault-generated:end -->` belongs to MindVault and is rewritten on rebuild.
- **Everything outside the markers is human text and is preserved verbatim** — write your
  own sections above or below the block freely. A test proves rebuild keeps them.
- Rebuild snapshots the map first, like every mutation.
- If the markers were deleted, rebuild appends a fresh generated block at the end (with a
  warning) rather than touching any existing text.

## Usage

- **Humans:** open the map in Obsidian; it is the project's table of contents and the
  graph view around it shows the project cluster.
- **Agents:** `mindvault_list_maps` + one `mindvault_read_note` of the map beats walking
  the vault with list/search calls. Rebuild after meaningful memory changes (the
  architecture-memory and organisation skills say when).
